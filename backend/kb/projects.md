# Projects

## What this covers
Projects are workspaces that group your documents and compliance runs together.

## Steps to create a project
1. Go to `/projects`.
2. Click **New Project**.
3. Enter a name (e.g. "SOC 2 Type II — 2026") and optional description.
4. Click **Create**. The project opens immediately.

## Steps to manage a project
- **Add documents** — click "Upload Document" inside the project.
- **Start a run** — click "Run Questionnaire" or go to `/run` and select this project.
- **View runs** — the project detail page lists all past runs.
- **Delete a project** — use the project settings menu. This deletes associated runs and documents.

## Common errors
- **"Project not found"** — the project may have been deleted, or you may not have access.
- **"Insufficient permissions"** — you need Owner, Admin, or Compliance Manager role to create or delete projects.

## Team access
Projects are visible to all members of your organization. Role-based access controls apply to actions like upload and delete.
